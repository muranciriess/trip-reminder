传统模式的痛点

  • • 用户旅行决策链路极其复杂
  • • 搜索引擎只能解决"点"的问题
  • • 早期 AI 聊天机器人无法形成交易闭环

飞猪需要的 Agent

  • • 懂行 — 理解旅行业务领域知识
  • • 能算 — 实时查询价格、班次
  • • 能做 — 完成交易闭环

三次技术跃迁

从地图工具到多智能体协作

1.0
地图工具时代

基于地图的行程规划工具

AI 仅用于解析导入的行程信息，核心是地图展示 + 手动编排。

局限：用户操作链路长，无法自动规划，AI 参与度低

2.0
RAG 时代

线路库 + 智能行程助手 (MVP)

采用 RAG 模式：基于 Query Processing 召回关联线路，由大模型总结输出。

建设高质量线路库

QP 召回 → 大模型总结

痛点：缺少交通/酒店实时数据，结构单一，无法形成完整方案

3.0
多智能体时代 当前版本

意图识别 Agent + N 个场景 Agent

构建 多智能体架构：一个意图识别 Agent 做路由，N 个场景 Agent（行程、交通、酒店、讲解等）协作完成任务。

意图识别 Agent（路由中枢）

行程规划 Agent

交通查询 Agent

酒店推荐 Agent

景点讲解 Agent

编排方式：COT + Prompt Chaining
